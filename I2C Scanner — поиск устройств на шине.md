## Схема подключения
- **Arduino UNO:**
  - A4 (SDA) → SDA датчика
  - A5 (SCL) → SCL датчика
  - 5V → VIN датчика (если датчик 5V-толерантный)
  - GND → GND
- **Подтяжки:** 4.7кΩ на SDA и SCL к 5V (опционально, если на модуле нет встроенных)

## Зависимости
- `Wire.h` (встроенная в Arduino IDE)

## Код
```cpp
#include <Arduino.h>
#include <Wire.h>

void setup() {
    Serial.begin(9600);
    delay(2000);  // Ждём открытия Serial Monitor
    
    Serial.println("=== I2C SCANNER ===");
    Wire.begin();
    Wire.setClock(100000);  // 100kHz (стандарт I2C)
    
    Serial.println("Начинаю сканирование...");
}

void loop() {
    byte count = 0;
    
    // Перебираем все возможные I2C адреса (1-126)
    for (byte i = 1; i < 127; i++) {
        Wire.beginTransmission(i);  // Пытаемся связаться
        byte error = Wire.endTransmission();  // Получаем ответ
        
        if (error == 0) {
            // Устройство ответило
            Serial.print("Найдено: 0x");
            if (i < 16) Serial.print("0");  // Добавляем ведущий ноль
            Serial.print(i, HEX);
            
            // Подсказка для известных устройств
            if (i == 0x29) Serial.print(" (VL53L0X)");
            if (i == 0x27 || i == 0x3F) Serial.print(" (LCD 1602 I2C)");
            if (i == 0x68) Serial.print(" (DS3231/MPU6050)");
            if (i == 0x76 || i == 0x77) Serial.print(" (BME280/BMP280)");
            
            Serial.println();            count++;
        }
        
        delay(10);  // Пауза между адресами (не обязательно, но стабилизирует)
    }
    
    // Если ничего не нашли — выводим чек-лист
    if (count == 0) {
        Serial.println("НЕ НАЙДЕНО! Проверь:");
        Serial.println("1. Pull-up резисторы 4.7кОм на SDA и SCL к 5V");
        Serial.println("2. SDA → A4, SCL → A5 (для Arduino UNO)");
        Serial.println("3. VIN → 5V (или 3.3V для 3.3V устройств)");
        Serial.println("4. GND → GND (общая земля)");
    }
    
    delay(5000);  // Пауза 5 сек перед следующим сканированием
}
```

## Что делает
- Перебирает все 126 возможных I2C адресов (1-126)
- Пытается связаться с каждым через `Wire.beginTransmission()`
- Если устройство отвечает (`error == 0`) — выводит его адрес в HEX
- Повторяет каждые 5 секунд
- Если ничего не найдено — выводит чек-лист типичных ошибок

## Логика работы
1. **`Wire.begin()`** — инициализирует I2C шину (Arduino UNO = master)
2. **`Wire.setClock(100000)`** — устанавливает частоту 100kHz (стандарт I2C)
3. **`Wire.beginTransmission(addr)`** — начинает передачу на адрес `addr`
4. **`Wire.endTransmission()`** — завершает передачу, возвращает код ошибки:
   - `0` = успех (устройство ответило)
   - `1` = переполнение буфера
   - `2` = NACK при передаче адреса
   - `3` = NACK при передаче данных
   - `4` = другая ошибка
5. **`Serial.print(i, HEX)`** — выводит число в шестнадцатеричном формате

## Нюансы
- **Адрес 0x29** — стандартный для VL53L0X (но может быть изменён через XSHUT пин)
- **Адрес 0x27 / 0x3F** — типичные для LCD 1602 с I2C адаптером (PCF8574)
- **Pull-up резисторы** — обязательны для I2C. Без них шина не работает. Некоторые модули имеют встроенные подтяжки (проверь схему модуля)
- **Напряжение** — Arduino UNO выдаёт 5V на SDA/SCL. Если датчик 3.3V (VL53L0X, BME280) — нужен level shifter или делитель напряжения
- **Длинные провода** — I2C не любит длинные провода. Держи SDA/SCL < 30см

## Типичные адреса устройств
| Устройство     | Адрес (HEX)   | Примечание                 |
| -------------- | ------------- | -------------------------- |
| VL53L0X        | 0x29          | Можно изменить через XSHUT |
| LCD 1602 I2C   | 0x27 или 0x3F | Зависит от PCF8574         |
| BME280         | 0x76 или 0x77 | Зависит от пина SDO        |
| MPU6050        | 0x68 или 0x69 | Зависит от пина AD0        |
| DS3231 (RTC)   | 0x68          | Фиксированный              |
| OLED 0.96" I2C | 0x3C или 0x3D | Зависит от модуля          |
